# 技术面试准备全攻略

## 技术面试的考察维度

技术面试通常从以下 4 个维度全面评估候选人：

1. **基础知识**（20-30%）：数据结构、算法、语言特性、操作系统、网络协议
2. **项目经验**（20-30%）：项目架构、技术选型、难点解决、业务理解
3. **编码能力**（30-40%）：手撕代码、代码质量、边界条件、时间复杂度
4. **系统设计**（10-20%）：架构思维、权衡取舍、扩展性（中高级岗位重点）

## 编程语言基础高频考点

### Python 高频面试题

**Q1: 解释 Python 的 GIL 及其影响**

GIL（全局解释器锁）是 CPython 的互斥锁，确保同一时刻只有一个线程执行 Python 字节码。

影响：
- CPU 密集型任务中多线程无法利用多核（需要用多进程）
- I/O 密集型任务中多线程仍然有效（I/O 等待时释放 GIL）
- 解决方案：multiprocessing 模块、concurrent.futures、asyncio

**Q2: 装饰器（Decorator）原理和应用**

装饰器本质是高阶函数，接受一个函数作为参数，返回一个新函数。

```python
def timer(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        print(f"{func.__name__} 耗时: {time.time()-start:.2f}s")
        return result
    return wrapper

@timer
def process_data():
    # 业务逻辑
    pass
```

常见应用场景：日志记录、权限验证、缓存（@lru_cache）、重试机制、性能监控。

**Q3: 生成器（Generator）vs 迭代器**

生成器是用 yield 关键字的函数，惰性求值，节省内存。
迭代器实现了 __iter__ 和 __next__ 协议的对象。
所有生成器都是迭代器，但不是所有迭代器都是生成器。

**Q4: Python 的深拷贝和浅拷贝**

- 赋值（=）：引用同一对象
- copy.copy()：浅拷贝，只拷贝顶层对象
- copy.deepcopy()：深拷贝，递归拷贝所有嵌套对象

**Q5: with 语句和上下文管理器**

通过 __enter__ 和 __exit__ 方法实现资源自动管理，常见于文件操作、数据库连接、锁等场景。

### Java/后端高频考点

**Q1: HashMap 底层原理（JDK8+）**

数组 + 链表 + 红黑树。默认初始容量 16，负载因子 0.75。
链表长度超过 8 且数组长度超过 64 时转为红黑树。
哈希冲突解决：链地址法。扩容时翻倍。

**Q2: Spring Boot 自动装配原理**

@SpringBootApplication 注解包含 @EnableAutoConfiguration，
通过 SpringFactoriesLoader 加载 META-INF/spring.factories 中的配置类，
按 @Conditional 条件判断是否生效。

**Q3: MySQL 索引原理**

B+ 树结构，所有数据存在叶子节点，叶子节点通过指针连接。
聚簇索引 vs 非聚簇索引（回表查询）、覆盖索引、最左前缀原则。

**Q4: Redis 常见数据结构和使用场景**

String（缓存、计数器）、Hash（用户信息）、List（消息队列）、Set（去重、标签）、ZSet（排行榜）。
缓存问题：缓存穿透（布隆过滤器）、缓存击穿（互斥锁）、缓存雪崩（随机过期时间）。

## 编码面试常见题型

### 必刷高频题分类

**数组与双指针**
- 两数之和（哈希表 / 双指针）
- 三数之和（排序 + 双指针）
- 盛最多水的容器（双指针）
- 合并两个有序数组（双指针）

**链表**
- 反转链表（迭代 / 递归）
- 链表中环的检测（快慢指针）
- 合并两个有序链表
- LRU 缓存（哈希表 + 双向链表）

**二叉树**
- 前中后序遍历（递归 / 迭代）
- 层序遍历（BFS）
- 二叉树的最大深度
- 翻转二叉树
- 最近公共祖先

**动态规划**
- 爬楼梯 / 斐波那契
- 最长递增子序列
- 0-1 背包问题
- 编辑距离
- 最长公共子序列

**排序算法**
- 快速排序（面试最常手撕的排序）
- 归并排序
- 堆排序

### 编码面试技巧

1. **先确认理解**：复述题目，确认输入输出和边界条件
2. **讨论方案**：先说思路，再写代码（不要上来就敲）
3. **分析复杂度**：时间复杂度 + 空间复杂度
4. **写完自测**：用示例数据走一遍代码
5. **优化思路**：如果还有时间，讨论是否有更优解
