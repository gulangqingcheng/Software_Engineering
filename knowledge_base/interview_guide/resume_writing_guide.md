# 简历撰写完全指南

## 简历的核心原则

简历的唯一目的是：**获得面试机会**。它不是自传，而是你的营销文案。面试官平均只花 6-10 秒扫描一份简历，因此必须做到：

1. **一页纸原则**：校招生简历严格控制在 1 页（A4），社招 1-2 页
2. **倒序排列**：最近的经历放最前面
3. **数据说话**：用量化结果替代模糊描述
4. **针对岗位**：不同岗位准备不同版本的简历

## 简历的核心模块

### 1. 个人信息（简洁明了）

必填：姓名、电话、邮箱、求职意向
可选：GitHub 主页、技术博客、LeetCode 链接
不需要：身份证号、籍贯、身高体重、政治面貌（除非国企）

### 2. 教育背景（校招生重点）

格式：学校名称 | 专业 | 学历 | 毕业时间

校招生建议补充：
- GPA（3.5+/4.0 才写）
- 核心课程（选 5-8 门与岗位相关的）
- 奖学金/竞赛奖项
- 排名（前 10% 才值得写）

### 3. 专业技能（分类清晰）

错误示范："熟悉 Java、Python、MySQL、Redis、Spring Boot、Vue..."

正确示范（分层次）：

```
编程语言：Python（熟练）、Java（熟悉）、JavaScript/TypeScript（熟悉）
后端框架：FastAPI、Flask、Spring Boot、Django
前端框架：Vue3、React 基础
数据库：MySQL（索引优化、分库分表）、Redis（缓存策略）、MongoDB
DevOps：Docker、Git、Linux 基础、CI/CD
AI/ML：PyTorch、LangChain、ChromaDB、Prompt Engineering
```

技能描述用词标准：
- **精通**：能从零造轮子，理解底层原理
- **熟练**：日常开发主力，能独立完成复杂需求
- **熟悉**：使用过，能完成基本开发
- **了解**：看过文档，写过 Demo

### 4. 项目经验（最核心的部分，占 40% 篇幅）

使用 **STAR 简缩版** 描述每个项目：

```
项目名称 | 技术栈 | 时间（月-月）
一句话概括项目（解决了什么问题，面向谁）
- 负责模块 A：使用 [技术] 实现了 [功能]，[量化成果]
- 负责模块 B：优化了 [指标]，[对比数据]
- 技术亮点：[独特的技术方案/创新点]
```

常见错误与修正：

| 错误写法 | 正确写法 |
|---------|---------|
| "优化了数据库查询性能" | "通过添加复合索引 + SQL 重写，将用户列表查询从 1.2s 优化到 50ms，QPS 提升 20 倍" |
| "开发了后台管理系统" | "独立负责后台管理系统的 RBAC 权限模块，基于 JWT + Redis 实现了细粒度的按钮级权限控制" |
| "使用了 Vue3" | "使用 Vue3 + Composition API + Pinia 重构前端，组件复用率提升 40%，首屏加载时间减少 60%" |
| "参与了 AI 面试助手项目" | "设计并实现了基于 RAG 的面试题生成 Agent，集成 ChromaDB + BGE Embedding，面试题相关性评分提升 35%" |

### 5. 实习经历（如有）

格式与项目经验类似，重点突出：
- 你在实习中独立完成的产出
- 你获得的技术成长
- 你得到的评价/认可

### 6. 荣誉奖项（按含金量排序）

国家级 > 省级 > 校级 > 院级
竞赛类 > 奖学金 > 学生干部

## 简历中的量化指标参考

| 岗位方向 | 可量化的指标 |
|---------|------------|
| 后端开发 | QPS、响应时间、并发量、可用性、数据库查询优化比 |
| 前端开发 | 首屏加载时间、Lighthouse 评分、组件复用率、包体积减少 |
| AI/算法 | 模型准确率、推理速度、训练数据量、评价指标（F1/AUC） |
| 全栈 | 完成功能数、用户量、DAU、开发周期缩短比例 |

## 校招 vs 社招简历区别

| 维度 | 校招生 | 社招生 |
|------|--------|--------|
| 核心重点 | 项目经历 + 教育背景 | 工作经历 + 项目成果 |
| 篇幅 | 严格 1 页 | 1-2 页 |
| 技能描述 | 基础扎实 + 学习能力 | 深度 + 广度 + 架构能力 |
| 亮点 | 竞赛获奖 + 开源贡献 | 业绩数据 + 技术影响力 |

## 简历自检清单

投递前逐项检查：

- [ ] 简历控制在 1 页（校招）或 1-2 页（社招）
- [ ] 没有错别字和格式不一致
- [ ] 项目经验有量化数据
- [ ] 技能描述有层次（熟练/熟悉/了解）
- [ ] 针对目标岗位做了定制
- [ ] PDF 格式导出（不要 Word）
- [ ] 文件命名规范：姓名-学校-岗位-电话.pdf
- [ ] 联系方式准确无误
